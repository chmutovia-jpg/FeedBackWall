# Demo Script

A 60-second walkthrough structure for a portfolio video, PM application, or interview demo.

## Structure

| Time | Focus | On screen |
|---|---|---|
| 0-10s | The problem | Landing hero and problem section |
| 10-20s | Product loop | How it works and feature cards |
| 20-35s | Public wall | BetaRoom wall, feedback question, feedback submission |
| 35-50s | Owner dashboard | Signals, filters, status change, maker response |
| 50-60s | Case study | Metrics, decisions, roadmap |

## Spoken Script

**0-10s**

> "When makers share an early project, feedback usually comes back as DMs, comments, and
> screenshots. Useful signal gets scattered before it becomes a decision."

**10-20s**

> "FeedbackWall gives that project one public wall. The maker asks one focused question, shares
> the link, and gets typed feedback that can be upvoted, triaged, and shipped."

**20-35s**

> "Here is a demo wall for BetaRoom. Visitors can understand the project, answer the maker's
> question, attach screenshots, and submit feedback without creating an account."

**35-50s**

> "On the owner side, the maker sees the queue as product signals. They can filter, search, move
> items through statuses, and add a response so visitors can see progress."

**50-60s**

> "The case study explains the product decisions: no AI, no billing, no complex analytics, just
> the smallest useful feedback loop from public link to shipped signal."

## Recording Notes

- Use the BetaRoom content from `docs/DEMO_DATA.md`.
- Capture at a desktop width around 1280px.
- Keep screenshots in the dark-only interface.
- If recording local fallback mode, clear `localStorage` first so votes and statuses start clean.
